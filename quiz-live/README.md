# QuizLive — онлайн-квизы в реальном времени (MVP)

Веб-приложение для проведения живых квизов: организатор создаёт квиз и запускает
игру, участники подключаются по 6-значному коду комнаты, отвечают на вопросы
в реальном времени и видят лидерборд по итогам.

## Возможности

**Аутентификация и роли** — регистрация/вход (JWT), роли «организатор» и «участник»

**Конструктор квизов** — категории, время на вопрос (общее и индивидуальное),
текстовые вопросы и вопросы с изображением, одиночный и множественный выбор

**Игра в реальном времени** — код комнаты, лобби, синхронный показ вопросов,
серверный таймер (вопрос доступен только во время демонстрации)

**Подсчёт результатов** — 500 баллов за верный ответ + до 500 бонуса за скорость,
промежуточный и итоговый лидерборд

**Личный кабинет** — история и статистика для участника; история игр
и статистика по каждому квизу для организатора

**Хранение данных** — SQLite: пользователи, квизы, вопросы, сессии, результаты

## Технологический стек

|Слой|Технология|
|-|-|
|Клиент|React 18 + Vite (SPA), react-router-dom|
|Сервер|Node.js + Express (REST API)|
|Real-time|Socket.IO (WebSocket)|
|БД|SQLite (better-sqlite3)|
|Аутентификация|JWT + bcryptjs|

## Запуск (режим разработки)

Требуется Node.js ≥ 18.

```bash
# Терминал 1 — сервер (http://localhost:3001)
cd server
npm install
npm run dev

# Терминал 2 — клиент (http://localhost:5173)
cd client
npm install
npm run dev
```

Откройте http://localhost:5173. Vite проксирует `/api` и `/socket.io` на сервер.

## Запуск (продакшен, один порт)

```bash
cd client \\\\\\\&\\\\\\\& npm install \\\\\\\&\\\\\\\& npm run build
cd ../server \\\\\\\&\\\\\\\& npm install \\\\\\\&\\\\\\\& npm start   # http://localhost:3001 раздаёт и клиент
```

## Как провести квиз

1. Зарегистрируйтесь как **организатор** → «+ Новый квиз» → добавьте вопросы → «Сохранить».
2. На карточке квиза нажмите **«Запустить»** — появится код комнаты.
3. Участники (регистрация с ролью **участник**) вводят код в своём кабинете.
4. Организатор нажимает «Начать квиз» и листает вопросы; сервер сам закрывает
приём ответов по таймеру и считает баллы.
5. По завершении все видят итоговый лидерборд; результаты сохраняются в кабинетах.

## Структура

```
server/src/
  index.js   — Express + Socket.IO, раздача собранного клиента
  db.js      — схема SQLite
  auth.js    — регистрация, вход, JWT-middleware
  routes.js  — REST: квизы, вопросы, история, статистика
  game.js    — игровой движок: комнаты, таймеры, баллы, лидерборд
client/src/
  pages/     — Auth, Dashboard, QuizEditor, Host, Play
  components/Game.jsx — таймер, карточка вопроса, лидерборд
```
